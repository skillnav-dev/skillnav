# ADR-002: 产品方向 — 从"目录"转型为"指南"

- **Status**: accepted
- **Date**: 2026-03-18
- **Context**: 产品走查 + 流量数据分析

## 背景

Umami 数据显示月访客 142 人，人均浏览 7.5 页（内容有价值），但 MCP 3,947 页仅占 1% PV。产品走查发现核心问题：SkillNav 是一个"数据库 + 翻译层"，缺乏编辑观点和用户引导。详情页是数据卡片（评分 6.5/10），不是决策依据。

## 决策

**从"数据目录"转型为"编辑指南"。**

### 三层内容架构

| 层级 | 内容 | 数量 | 投入 | 目标 |
|------|------|------|------|------|
| 编辑层 | "最佳 X" 指南、评测、教程 | 10-20 篇 | 高（手写） | 引流 + 信任 |
| 策展层 | S/A 级工具完整页 | ~500 | 中（模板+审核） | 留存 + SEO |
| 索引层 | B-tier 长尾 | ~3,500 | 低（自动化, noindex） | 长尾 SEO |

### 实施阶段

1. 场景化入口（首页 "你想做什么"）✅
2. 做厚 S-tier 详情页
3. 编辑原创内容
4. 分发节奏

### Skills 架构变更

仓库（GitHub repo）为一级实体，Skill 为二级实体。默认 `/skills` 展示 7 个仓库卡片，点击进入仓库详情。

## 不做的事

- 不再扩大工具数量（3,947 够了）
- 不做新功能（全站搜索、用户账号）
- 不扩展英文内容
- 不投入 B-tier 页面丰富化

## 后果

- 短期内产品形态明显变化（首页、Skills 列表）
- 需要编辑原创能力（非纯自动化）
- SEO 策略从"数量覆盖"转为"质量集中"
