## 一、战略定位

```
核心原则：skillnav.dev 是内容沉淀地，社交平台是分发管道
分发目标：品牌认知 → 站点回流 → 用户留存
```

**一鱼多吃**：一次创作 → 多平台适配分发，不为每个平台单独创作。

**阮一峰 + idoubi 混合模型**：
- 阮一峰的启示：极简渠道、固定节奏、不搞社群、周刊可持续
- idoubi 的启示：Build in Public、即刻+X 建个人 IP、产品即内容

## 二、平台优先级

### 第一梯队 — Must（立即启动）

| 平台 | 定位 | 核心价值 | 月投入 |
|------|------|---------|--------|
| **X (Twitter)** | 一手分发 + 个人 IP | AI 资讯上游平台，中文开发者高度活跃 | ~8h + $8 |
| **微信公众号** | 私域沉淀 + 品牌触达 | 中文技术内容最大流量入口 | ~4h + ¥300/年 |
| **知乎** | 长尾 SEO | 百度搜索权重极高，AI 话题活跃 | ~3h |
| **即刻** | 社区互动 + 个人 IP | AI 从业者聚集地，零成本碎片发布 | ~2h（碎片） |

### 第二梯队 — Nice-to-have（第一梯队稳定后加入）

| 平台 | 定位 | 核心价值 | 启动条件 |
|------|------|---------|---------|
| **掘金** | 开发者社区覆盖 | Markdown 直接同步，零改编成本 | 第一梯队运行 1 个月后 |
| **CSDN** | 百度 SEO 占位 | 仅自动同步，不运营 | 有多平台同步工具后 |

### 不做

| 平台 | 理由 |
|------|------|
| **B 站** | 视频制作成本过高，1 人团队不可持续 |
| **小红书** | 严禁外链无法引流 + 图片制作成本高 + 偏消费调性；保持观察，AI 开发者社区成熟后重新评估 |
| **公众号社群** | 维护成本极高，阮一峰经验证明不值得 |

## 三、平台分策略

### 3.1 X (Twitter)

**账号设置**：
```
名称: SkillNav
Handle: @skillnav_dev
Bio: "AI 智能体工具站 | Skills · MCP · 实战资讯 | 每周精选不水 | skillnav.dev"
订阅: X Premium ($8/月) — 回复置顶 + 6x 曝光提升
```

**人设**：有主编人格的品牌号。专业但有观点，不写客服体。

**差异化**：聚焦 Skills + MCP + Agent 工具实操，区别于宝玉（翻译解读）和歸藏（AIGC 周刊）。

**内容节奏**：

| 内容 | 形式 | 频率 |
|------|------|------|
| 翻译资讯精选 | 单条推文：一句话总结 + 观点 + 链接 | 每天 1-2 条 |
| 周刊摘要 | 7 条 Thread：hook + 5 条精选 + CTA | 每周一 |
| 工具评测 | Thread (5-10 条) + 截图/GIF | 每周 1 篇 |
| 行业快评 | Quote Tweet 大号推文 + 中文解读 | 有热点就发 |
| 互动帖 | 提问/投票 | 每周 1-2 条 |

**发布时间**（中文开发者受众）：
- 工作日 9:00-11:00 / 12:00-13:00 / 21:00-23:00
- 周刊 Thread：周一上午 9:00-10:00

**冷启动策略**：
1. 开通 Premium + 完善资料
2. 准备 3 个"支柱 Thread"作为起手式
3. 每天评论大号推文 15-20 条（宝玉/歸藏/vista8/Anthropic 官方）
4. 评论要有实质内容（补充工具推荐、实操经验），而非"写得好"

**增长预期**：1 月 100-300 粉 → 3 月 1K → 6 月 3K → 12 月 10K

**算法要点**：
- 作者回复评论权重 75x，必须经营评论区
- 转发 20x > 回复 13.5x > 点赞 1x → 写引发讨论的内容
- Grok AI 偏好积极/建设性内容，技术内容天然被偏好
- 外链降权已取消，但 Premium 账号才有效果

### 3.2 微信公众号

**账号类型**：订阅号（建议企业主体 + 微信认证 ¥300/年，确保 API 权限）

**发布规则（2025-2026 最新）**：
- **群发**（开启群发通知）：每天 1 次，出现在订阅号消息列表
- **发布**（关闭群发通知）：**不限次数**，不出现在消息列表，但可被算法推荐到流量池
- 即：随时可发布，不受次数限制；群发是用于重点内容的"主动推送"

**内容节奏**：

| 内容 | 发送方式 | 频率 | 原创声明 |
|------|---------|------|---------|
| SkillNav Weekly 精简版 | **群发**（头条） | 每周一 | 可声明 |
| 深度评测/课程导读 | **群发**（头条） | 有内容时 | 可声明 |
| 月度盘点 | **群发**（头条） | 每月初 | 可声明 |
| 翻译资讯精选 | **发布**（不限次数） | 随时，有好内容就发 | **不声明**（翻译不允许） |

**周刊公众号版改编要点**：
- 控制在 2000-3000 字，保证完读率
- 去掉安装命令，保留推荐理由和核心观点
- 文末 "阅读原文" → `skillnav.dev` 对应文章页

**排版工具链**：
- MVP：mdnice（在线 Markdown → 公众号富文本，一键复制）
- 进阶：wenyan-mcp（Markdown → 排版 → 推送草稿到公众号后台）

**引流策略**（正文不能放外链）：
- "阅读原文"链接 → skillnav.dev（预期 3-8% 点击率）
- 文末引导文案："完整工具列表/代码请访问 skillnav.dev"
- 底部菜单栏设置 "工具导航" → skillnav.dev
- 关注后自动回复站点介绍 + 链接

**关键规则**：
- 翻译内容**不可**声明原创（否则被投诉封号）
- AI 生成内容需确保有充分人工编辑痕迹
- 群发保持每周 1-2 次，不要日更（算法推荐周期考虑）

### 3.3 知乎

**账号形式**：个人号 + 专栏

**内容策略**：

| 内容 | 形式 | 频率 |
|------|------|------|
| 公众号长文同步 | 专栏文章 | 跟公众号节奏 |
| 热门问题回答 | 问答 | 每周 1-2 个 |
| 工具推荐/评测 | 专栏文章 | 每周 1 篇 |

**高价值问答选题方向**：
- "Claude Code 有哪些好用的 Skill？"
- "MCP 是什么？怎么用？"
- "AI 编程工具怎么选？"

**引流方式**：
- 回答中自然提及 "更多工具推荐见 skillnav.dev"
- 专栏文章底部放站点链接

**核心价值**：百度搜索权重极高，一篇优质回答可持续数月带来流量。

### 3.4 即刻

**账号定位**：AI 工具达人 + Build in Public

**内容节奏**：

| 内容 | 频率 |
|------|------|
| 翻译资讯要点提炼 | 每天 1 条（碎片时间） |
| 产品动态（Build in Public） | 有更新就发 |
| 工具推荐短评 | 每周 2-3 条 |

**目标圈子**：AI 探索站、独立开发者、Claude Code

**特点**：不提供 SEO 价值，但提供圈子认同感和个人 IP。碎片时间发布，成本极低。

## 四、内容适配矩阵

一篇内容如何"一鱼多吃"：

```
skillnav.dev 文章（完整版 Markdown）
    │
    ├─→ X: 单条推文摘要 + 链接（或 Thread 版）
    ├─→ 公众号: 精简版 + mdnice 排版 + "阅读原文"
    ├─→ 知乎: 同公众号版，Markdown 直接发专栏
    ├─→ 即刻: 1-2 句核心观点 + 评论
    ├─→ 掘金: Markdown 直接粘贴
    └─→ CSDN: 自动同步（工具代发）
```

**各平台改编规则**：

| 平台 | 字数 | 格式 | 标题风格 | 外链 |
|------|------|------|---------|------|
| X | ≤250字/条 | 纯文字 + 截图 | 有观点、引发讨论 | 可放（Premium） |
| 公众号 | 2000-3000 | 富文本（mdnice） | 具体 + 数据驱动 | 仅"阅读原文" |
| 知乎 | 不限 | Markdown | 专业客观 | 正文可放 |
| 即刻 | ≤500字 | 短文 + 图片 | 口语化、轻松 | 可放 |
| 掘金 | 不限 | Markdown 原生 | 技术标题 | 正文可放 |

**内容战略 2.0 各层级的分发映射**：

| 内容层级 | skillnav.dev | X | 公众号 | 知乎 | 即刻 |
|---------|-------------|---|--------|------|------|
| 日更 RSS 翻译 | 素材库 | 精选 1-2 条推文 | 精选发布（非群发） | — | 要点提炼 |
| 周刊 | 完整版 | Thread 摘要版 | 群发精简版 | 专栏同步 | 要点 |
| 深度评测 | 完整版 | Thread + 截图 | 群发头条 | 专栏 | 短评 |
| 月度盘点 | 完整版 | Thread | 群发头条 | 专栏 | 要点 |

## 五、自动化管线

### 5.1 自动化程度分级

| 环节 | 自动化程度 | 工具/方案 |
|------|-----------|----------|
| RSS 采集 + 翻译 | **全自动** | 已有 `sync-articles.mjs` |
| 文章 → X 推文草稿 | **半自动** | n8n + LLM 生成 → 人工审批 |
| 文章 → 公众号排版 | **半自动** | mdnice 手动 → 后期 wenyan-mcp |
| 文章 → 知乎/掘金 | **手动** | Markdown 直接复制 |
| 即刻动态 | **手动** | 碎片时间随手发 |
| 多平台同步 | **后期** | OpenWrite / Wechatsync 插件 |

### 5.2 X 自动化管线（n8n）

```
Supabase 新文章
    ↓ n8n Webhook / 定时轮询
LLM 生成推文草稿 (GPT proxy)
    ├─ 资讯翻译 → 单条推文
    ├─ 深度文章 → Thread (5-7 条)
    └─ 周刊 → Thread (精选 5 条)
    ↓
Slack/Telegram 人工审批
    ↓
X API (Free Tier) 发布
    ↓
记录到 Supabase (追踪效果)
```

**成本**：n8n 自托管 $0 + X API Free Tier $0 + LLM ~$0.01/篇 ≈ **$0/月**

### 5.3 公众号自动化路径

```
Phase 1 (立即): Markdown → mdnice → 手动复制到微信后台 (~15min/篇)
Phase 2 (2-3月后): wenyan-mcp → 自动推草稿到微信后台 → 人工审核群发
Phase 3 (6月后): GitHub Actions 定时生成 + 推送草稿
```

**前提**：企业认证后才有 API 发布权限（个人号只能创建草稿）。

## 六、每周 SOP

### 工时预算：6-8 小时/周

| 日 | 工作 | 耗时 | 平台 |
|----|------|------|------|
| 周一 | 发布周刊：站内完整版 + X Thread + 公众号精简版 + 知乎 | 2h | 全平台 |
| 周二 | 浏览翻译资讯，选 1-2 条精华 → X 推文 + 即刻 | 30min | X + 即刻 |
| 周三 | X 大号评论互动 15-20 条 + 即刻动态 | 30min | X + 即刻 |
| 周四 | 精选翻译资讯 → X + 公众号发布（非群发）| 30min | X + 公众号 |
| 周五 | 知乎回答 1 个热门问题 + X 评论互动 | 45min | 知乎 + X |
| 周末 | 深度评测/工具推荐 → 全平台分发（有精力时） | 1-2h | 全平台 |
| 每天 | X 碎片评论 + 即刻碎片动态 | 15min | X + 即刻 |

### 周一发布日详细流程

```
1. 生成周刊完整版 → 发布到 skillnav.dev          (10min)
2. 提炼 Thread 版 → 发 X (7条: hook+5精选+CTA)    (20min)
3. Markdown → mdnice → 复制到公众号后台            (15min)
   → 设置"阅读原文" → 预览 → 群发
4. 同一 Markdown → 粘贴到知乎专栏                  (5min)
5. 即刻发一条周刊要点速览                          (5min)
```

## 十一、关键决策记录

1. **小红书暂不做**：严禁外链无法引流 + 图片制作成本高，观察 AI 开发者社区成熟度后重新评估
2. **B 站不做**：视频制作成本过高
3. **不维护社群**：参考阮一峰经验，社群维护成本远大于收益
4. **公众号选订阅号**：群发每天 1 次 + 发布不限次数；服务号仅月 4 次群发
5. **X 中文为主**：目标用户是中文开发者，术语保留英文
6. **MVP 先手动**：不在自动化上过度投入，先验证内容-市场匹配
7. **周一为统一发布日**：所有平台同一天发周刊，降低认知负担
