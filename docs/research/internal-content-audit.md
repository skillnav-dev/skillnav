## 三、翻译管线架构

### 3.1 自适应翻译策略

```
≤15K 字符  → 单次调用 (translateArticleSingle)
15K-50K    → 分块翻译 (translateArticleChunked)
>50K       → 精华摘要 (summarizeArticle)
```

### 3.2 LLM Provider 支持

5 个 provider：DeepSeek / Gemini / Anthropic / OpenAI / GPT (proxy)

CI 环境使用 `LLM_PROVIDER=gpt`（GitHub Actions 直连代理 gmn.chuangzuoli.com）

### 3.3 质量控制机制

| 维度 | 机制 | 状态 |
|------|------|------|
| 去重 | `source_url` 唯一约束 | ✅ |
| 长度过滤 | 内容 <500 字 skip | ✅ |
| 关键词过滤 | RELEVANCE_KEYWORDS (45 个) | ✅ |
| JSON 解析 | `sanitizeJsonString()` + 二级降级 | ✅ |
| Feed 截断 | 100 items/feed | ✅ |
| 自动评分 | relevance_score 1-5 | ✅ |

### 3.4 评分处置规则

```
relevance_score: 4-5 → status: published
relevance_score: 3   → status: published
relevance_score: 2   → status: draft (待审)
relevance_score: 1   → status: hidden
```
