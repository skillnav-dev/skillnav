# 竞品 UI/UX 设计分析

## 分析概要

- **分析日期**: 2026-03-08
- **竞品数量**: 8 个（5 个直接竞品 + 3 个间接竞品/参考）
- **分析方法**: WebFetch 直接抓取 + WebSearch 多轮搜索 + 项目已有竞品情报（MEMORY.md）
- **SkillNav 定位**: 中文开发者的 AI 智能体工具站（Skills · MCP · 实战资讯）

### 关键发现

1. **搜索为王**: 所有成功的工具导航站都将搜索框置于首屏核心位置（mcp.so、ToolAI、Glama）
2. **Tab 分流**: 头部竞品普遍使用 Tab 系统分流内容（Today/Featured/Latest/Official 等）
3. **卡片密度差异大**: MCP 目录站偏高密度信息卡片，媒体站偏大图 + 标题
4. **中文站普遍缺乏暗色模式**: ai-bot.cn、36kr、机器之心均无暗色模式，daily.dev 是典范
5. **编辑精选是差异化关键**: Glama 的质量徽章、HelloGitHub 的月刊精选模式、daily.dev 的个性化推荐，都比纯数量堆砌更有价值
6. **SEO 结构化数据**: Glama 使用 SoftwareApplication JSON-LD，mcp.so 作为开源模板项目 SEO 较弱

## 逐一竞品分析

### 1. mcp.so

- **定位**: MCP Server 导航站，最大的第三方 MCP 市场（18,250+ servers），月访 1M+
- **创建者**: idoubi，从 gpts.works（GPTs 第三方导航站）模板改造而来
- **技术栈**: Next.js + Supabase + Vercel（与 SkillNav 几乎相同）

#### 首页设计
- **Hero 区域**: 简洁标题 "Find Awesome MCP Servers and Clients"，直接传达价值主张
- **首屏效率**: 高。标题 + 搜索 + Tab 分类（Today/Featured/Latest/Clients/Hosted/Official）+ 卡片网格，一屏内完成"理解站点 → 开始浏览"的闭环
- **信息密度**: 中等偏高，卡片紧凑但不拥挤
- **赞助位**: 首页有赞助 Server 展示，平衡了商业化和用户体验

#### 导航系统
- **主导航**: 浮动侧边栏（可折叠），包含 Home/Servers/Clients/Categories/Tags/Feed
- **多路径发现**: 支持分类浏览、标签筛选、搜索三种发现方式
- **社交入口**: X/GitHub/Discord/Email 社区链接集成在导航中

#### 列表/详情页
- **卡片样式**: 极简风格 — 名称 + 简短描述 + 创建者信息
- **网格密度**: 标准网格，每行 3-4 个卡片
- **详情页**: 单独页面展示 Server 详细信息、安装方式、功能说明
- **分页**: 支持分页浏览大量结果

#### 视觉语言
- **配色**: 中性色系，以可读性为优先
- **字体**: 标准 sans-serif，通过字重和大小建立层次
- **图标**: Remix Icon 库
- **品牌感**: 简约技术风，"MCP.so" logo 简洁

#### 亮点/借鉴
- **Tab 分类系统**: Today/Featured/Latest/Hosted/Official 的分流方式非常高效
- **FAQ 教育区**: 直接在首页解释 MCP 概念，降低新用户门槛
- **开源模板**: 代码开源（chatmcp/mcp-directory），可直接参考实现
- **快速上线**: idoubi 2 小时从 gpts.works 改造而来，证明模板复用的威力

#### 问题/避免
- **卡片信息不足**: 缺少 Star 数、下载量等量化指标，难以判断质量
- **无质量评级**: 18,000+ servers 无质量筛选，用户需要自行判断
- **SEO 较弱**: 作为模板站，结构化数据不够丰富
- **无中文支持**: 纯英文界面，中文开发者体验差
- **视觉同质化**: 和大量同类导航站视觉上难以区分

#### 截图要点
- 首屏：左侧可折叠侧边栏 + 右侧主内容区（标题 + Tab 栏 + 卡片网格）
- 卡片：白色背景、圆角、阴影轻微、名称粗体 + 描述灰色文本
- 底部：Footer 分组链接（Explore/Settings/Legal）

### 2. ai-bot.cn（AI工具集）

- **定位**: 中文 AI 工具导航，1000+ 工具收录，国内最具影响力的 AI 工具导航站之一
- **目标用户**: 专业人士、创意工作者、AI 技术爱好者

#### 首页设计
- **Hero 区域**: 简洁大气，直接展示核心价值"AI工具集导航"
- **内容组织**: 侧边栏分类 + 主内容区工具卡片
- **信息密度**: 高 — 首页即展示大量工具卡片和分类入口
- **首屏效率**: 较高，用户可快速定位到感兴趣的分类

#### 导航系统
- **分类体系**: 明确的功能分类 — 写作/图像/视频生成/编程开发/创意设计等
- **侧边栏**: 详细分类导航，左侧固定
- **搜索**: 支持快速筛选和跳转
- **特色板块**: AI应用集（APP 下载）、每日AI快讯、AI学习资源

#### 列表/详情页
- **卡片样式**: 工具名称 + 简介 + 分类标签 + 直达链接
- **详情页**: 工具详情 + 同类推荐（下方展示相似工具）
- **筛选维度**: 按功能分类、免费/付费、热门/最新

#### 视觉语言
- **配色**: 浅色系，简洁大方
- **品牌感**: "AI工具集" 中文品牌明确
- **风格**: 信息实用型，不追求炫酷
- **加载速度**: 快速稳定（用户评价）

#### 亮点/借鉴
- **同类推荐**: 详情页下方的"同类工具推荐"是非常好的交叉引流设计
- **每日AI快讯**: 工具导航 + 资讯内容的融合，和 SkillNav 的定位非常接近
- **中文优先**: 完全面向中文用户，语言和内容都本地化
- **APP集合**: 区分 Web 工具和可下载应用，分类维度创新

#### 问题/避免
- **无暗色模式**: 纯浅色界面，缺少主题切换
- **官网声明**: 需要声明"唯一官网地址"，说明存在大量仿冒站，品牌保护不足
- **信息密度过高**: 首页同时展示太多内容，可能造成选择疲劳
- **社区互动弱**: 缺少用户评论、评分等 UGC 内容

#### 截图要点
- 首屏：顶部导航 + 左侧分类侧边栏 + 右侧工具卡片网格
- 特色：每个工具卡片有功能图标 + 中文名称 + 简短描述 + 直达按钮
- 底部：每日AI快讯板块 + 推荐文章

### 3. toolai.io（ToolAI）

- **定位**: 全球 AI 工具目录和效率手册，26,871 个 AI 工具，459 个分类
- **规模**: 远超同类站点的收录量

#### 首页设计
- **Hero 区域**: "实用 AI 应用目录与效率手册"为核心口号
- **首屏效率**: 中等 — 分类入口众多，但核心行动路径清晰
- **信息密度**: 极高 — 近 27,000 个工具需要强大的信息架构支撑

#### 导航系统
- **分类体系**: 459 个分类，包括聊天机器人(6,280)、办公效率(22,310)、图像(8,403)、编程(8,621)、视频(4,518)等
- **多维筛选**: 按用途/免费/付费/热门/最新/地区/来源排名
- **搜索**: 标签化搜索支持
- **特色频道**: 免费工具区、Chrome 扩展、App 市场、Discord 社区、GPT Store

#### 列表/详情页
- **卡片样式**: 工具名称 + 简介 + 分类标签 + 收藏/使用量指标
- **排名系统**: 按月访量、地区、来源等多维度排名
- **每日更新**: 最新 AI 工具每日更新展示

#### 视觉语言
- **配色**: 现代简洁风格
- **品牌感**: "ToolAI" 国际化品牌
- **风格**: 数据驱动型，强调数量和覆盖面

#### 亮点/借鉴
- **多维排名**: 月访量/地区/来源的排名系统，给用户多角度评估工具质量的能力
- **每日更新**: 保持新鲜感，让用户有理由回访
- **GPT Store 集成**: 将 GPTs 也纳入收录，扩大覆盖面
- **Chrome 扩展/Discord 社区频道**: 将工具的不同形态分别展示

#### 问题/避免
- **数量压倒质量**: 26,871 个工具，筛选难度极大
- **分类过于庞大**: 459 个分类让用户选择成本很高
- **信息过载**: 首页同时呈现太多维度的内容
- **缺乏深度评测**: 以收录和聚合为主，缺少编辑观点

#### 截图要点
- 首屏：顶部搜索 + 分类标签云 + 工具卡片网格
- 特色：每个分类旁标注工具数量（如"编程 8,621"）
- 排名：独立的排名页面，按月访量/地区/来源排序

### 4. HelloGitHub

- **定位**: 有趣的开源项目推荐社区，GitHub 96k Stars，2016 年持续更新至今
- **创始人**: 知乎知名技术贡献者（620 篇文章，42,000 赞）

#### 首页设计
- **Hero 区域**: "分享 GitHub 上有趣、入门级的开源项目"
- **内容组织**: 月刊模式 — 每月 28 日发布新一期精选
- **信息密度**: 中等 — 每期精选数量有限，降低信息过载
- **首屏效率**: 高 — 最新一期月刊 + 热门项目直接呈现

#### 导航系统
- **主导航**: 首页/月刊/标签/排行/搜索
- **标签系统**: 按技术标签分类（Static Website、Web Application、推荐系统等）
- **月刊归档**: 按期数浏览历史推荐

#### 列表/详情页
- **卡片样式**: 项目名称 + 描述 + Star 数 + 语言标签 + 预览图
- **月刊式排版**: 每期按语言/类别分组，每个项目配详细中文描述
- **详情页**: 直接链接到 GitHub 仓库

#### 视觉语言
- **配色**: 清新简洁，白色为主
- **品牌感**: "HelloGitHub" 亲切友好的品牌调性
- **风格**: 社区感强，非商业化气质
- **图标**: GitHub 风格

#### 亮点/借鉴
- **月刊模式 = 内容节奏**: 每月一期的固定节奏，培养用户回访习惯
- **入门友好**: 明确标注"入门级"，降低开发者的心理门槛
- **深度中文描述**: 不是简单翻译 README，而是用通俗中文介绍每个项目的亮点
- **社区运营**: 知乎/公众号/GitHub 多平台联动
- **精选而非大全**: 每期精选而非大规模收录，质量 > 数量

#### 问题/避免
- **更新频率低**: 月更一次，对于快速变化的 AI 领域可能不够及时
- **缺少实时搜索**: 以浏览为主的发现方式，搜索体验相对弱
- **移动端适配**: 早期设计，移动端体验一般
- **无工具安装/使用引导**: 纯推荐，不提供一键安装或使用教程

#### 截图要点
- 首屏：最新一期月刊标题 + 精选项目列表
- 特色：每个项目配有中文描述段落 + GitHub Star 徽章 + 语言标签
- 月刊列表：按期数排列，封面图 + 标题 + 日期

### 5. glama.ai

- **定位**: MCP Server 目录 + 托管平台，17,900+ servers，提供质量评级和托管服务
- **差异化**: 不仅是目录，还提供 MCP 托管、API 网关和质量评估

#### 首页设计
- **Hero 区域**: MCP 平台定位，搜索为核心入口
- **首屏效率**: 高 — 搜索 + 分类 + 排序选项一步到位
- **信息密度**: 高但有序 — 通过多维筛选降低认知负担

#### 导航系统
- **搜索系统**: 支持"Deep Search"语义搜索 + 多种排序（搜索相关性/最近使用/添加日期/更新日期/周下载量/GitHub Stars/近期 Stars）
- **分类过滤**: Remote/Python/Developer Tools/TypeScript/Databases/RAG Systems/Autonomous Agents/Code Execution/Agent Orchestration 等 25+ 分类
- **特色入口**: "Add Server" 用户提交入口

#### 列表/详情页
- **卡片样式**: 名称 + 描述 + 质量徽章 + 使用统计 + 平台标签
- **质量徽章系统**: 核心差异化 — 基于安全性/兼容性/易用性自动评级
- **详情页**: 功能描述 + 安装指南 + Schema 页面 + 评分页面（Score）
- **API 访问**: REST API 提供所有 Server 信息（`/api/mcp/v1/servers/`）

#### 视觉语言
- **配色**: 专业深色系偏向，技术感强
- **品牌感**: "Glama" 现代科技品牌
- **风格**: 数据密集但层次清晰
- **结构化数据**: SoftwareApplication JSON-LD schema（SEO 最佳实践）

#### 亮点/借鉴
- **质量徽章系统**: 自动扫描和评级（安全性/兼容性/易用性），这是 SkillNav "Wirecutter for AI Agent Skills" 定位的完美参考
- **Server 认领**: 作者可通过 GitHub 认证认领自己的 Server，增加可信度
- **SoftwareApplication Schema**: SEO 最佳实践，Google 富媒体搜索结果
- **多维排序**: 7 种排序方式满足不同用户需求
- **MCP API**: 开放 API 供第三方使用，生态思维

#### 问题/避免
- **无中文支持**: 纯英文界面
- **功能过多**: 对新用户来说，过多的排序和筛选选项可能造成困惑
- **分类膨胀**: 25+ 分类标签，部分交叉重叠
- **托管功能复杂**: 托管 + 目录混合，定位模糊

#### 截图要点
- 首屏：搜索栏（Deep Search 模式）+ 排序下拉 + 分类标签云
- 卡片：带质量徽章（绿色/黄色/红色）+ 下载统计 + Star 数
- 详情页：左侧主内容 + 右侧安装指南 + 底部 Schema 信息

### 6. daily.dev（间接竞品/参考）

- **定位**: 开发者新闻聚合平台，"The homepage every developer deserves"
- **形态**: 浏览器扩展 + Web 平台，将新标签页变为个性化技术资讯 feed
- **技术**: 开源，免费，1,300+ 技术新闻源

#### 首页设计
- **Landing Page**: 面向未注册用户的营销页面，展示核心功能和用户评价
- **Feed 页面**: 登录后的主界面是个性化的文章流
- **首屏效率**: 极高 — 新标签页即展示个性化内容，零操作成本
- **信息密度**: 可调节 — 用户可选择卡片/列表布局、不同间距

#### 导航系统
- **Feed 定制**: 基于标签（#react, #python 等）和新闻源的个性化推荐
- **快捷方式**: 支持用户自定义快捷链接（可导入 Chrome 最常访问）
- **Squads**: 开发者社区小组，围绕共同兴趣聚合
- **Career Mode**: 求职相关功能

#### 列表/详情页
- **双布局模式**: 卡片视图 / 列表视图 可切换
- **卡片样式**: 文章标题 + 来源 + 阅读时间 + 点赞/评论数 + 预览图
- **间距可调**: 用户可调整卡片间距
- **Companion Widget**: 浏览器侧边栏小组件

#### 视觉语言
- **主题**: 暗色模式（默认）+ 浅色模式可切换
- **配色**: 深色背景偏蓝调（部分用户反馈"太蓝"，希望纯灰色）
- **品牌感**: 现代开发者工具风格，"daily" 品牌简洁有力
- **动画**: 适度的微动画，不过度干扰

#### 亮点/借鉴
- **新标签页模式**: 将内容消费嵌入浏览器日常操作，极致的触达频率
- **个性化引擎**: 基于用户兴趣标签自动调整推荐，越用越精准
- **双布局切换**: 卡片 vs 列表，满足不同浏览偏好
- **社区游戏化**: 评论/互动的轻度游戏化，鼓励参与但不过度
- **开源 + 免费**: 商业模式通过 Plus 会员（去广告/高级功能）而非限制免费功能

#### 问题/避免
- **暗色模式过蓝**: 用户反馈，纯灰色可能更适合长时间阅读
- **信息泛化**: 1,300+ 源可能导致内容质量参差不齐
- **初始设置成本**: 需要配置兴趣标签才能获得好体验
- **非目录型**: 不适用于工具发现场景，更适合资讯消费

#### 截图要点
- 新标签页：深色背景 + 个性化文章卡片网格 + 左侧标签侧边栏
- 卡片：预览图 + 标题 + 来源图标 + 阅读时间 + 互动指标
- 设置：布局切换（卡片/列表）+ 主题切换 + 间距调节

### 7. 36kr.com/AI（36氪 AI 频道）

- **定位**: 中国头部科技商业媒体，Nasdaq 上市（KRKR），日均 PV 600 万+
- **AI 频道**: 人工智能作为 20+ 垂直频道之一

#### 首页设计
- **Hero 区域**: 新闻信息流为主，无独立 Hero — 典型新闻媒体布局
- **内容组织**: 时间线 + 编辑推荐 + 分频道入口
- **信息密度**: 极高 — 媒体站标准，每日发布大量文章
- **首屏效率**: 中等 — 需要用户主动找到 AI 频道入口

#### 导航系统
- **主导航**: 创业/科技/汽车/消费/AI/新能源等 20+ 垂直频道
- **搜索**: 顶部搜索栏
- **快讯**: 实时快讯流
- **视频**: 36Kr TV 独立频道

#### 列表/详情页
- **文章卡片**: 大标题 + 摘要 + 封面图 + 作者 + 时间
- **详情页**: 标准新闻文章布局 — 标题/作者/时间/正文/相关推荐
- **数据库**: JingData 创业公司数据库（融资轮次/投资人）

#### 视觉语言
- **配色**: 红色主品牌色 + 白色背景
- **品牌感**: 成熟商业媒体气质，"36氪" 品牌辨识度高
- **字体**: 标准中文新闻字体
- **风格**: 传统新闻门户风格，功能优先于美感

#### 亮点/借鉴
- **快讯模式**: 实时快讯流适合 AI 领域的快速动态追踪

## SkillNav 可借鉴的 Top 10 设计决策

### 1. 搜索为王 — 首屏核心搜索体验（参考: mcp.so, Glama）
当前 SkillNav 已有 toolbar 搜索，但可以进一步强化。Glama 的 "Deep Search" 语义搜索和 mcp.so 的首屏大搜索框都证明：对于工具发现场景，搜索必须是最突出的 UI 元素。**建议**: 首页 Hero 区域增加全站搜索入口（跨 Skills + Articles），而非仅在列表页提供。

### 2. Tab 分流系统 — 多视角浏览（参考: mcp.so）
mcp.so 的 Today/Featured/Latest/Hosted/Official Tab 系统非常高效，用户一键切换不同维度。SkillNav 当前已有 featured/latest/all Tab，可以扩展为包含"平台"维度（Claude/Codex/Universal）和"精选/最新/全部"的组合 Tab。

### 3. 质量徽章系统 — 可视化信任（参考: Glama）
Glama 的质量徽章（基于安全性/兼容性/易用性自动评级）是最大的差异化。SkillNav 已有 `quality_tier` (A/B/C) 和 `editor_comment_zh`，应将其可视化为前端徽章（金/银/铜牌或星级），配合编辑短评，打造 "Wirecutter for AI Skills" 的体感。

### 4. 同类推荐引流（参考: ai-bot.cn）
ai-bot.cn 详情页底部的"同类工具推荐"是提升页面停留和浏览深度的利器。SkillNav Skill 详情页可在底部增加"相同分类的推荐 Skills"和"相关文章"交叉引流区域。

### 5. 月刊/精选节奏感（参考: HelloGitHub）
HelloGitHub 每月 28 日发布精选期刊的模式创造了"期待感"和回访节奏。SkillNav 可考虑"每周 AI Agent 工具精选"或"每月 Skill 评测榜"等固定节奏内容，配合 Newsletter 推送。

### 6. 暗色模式做好（参考: daily.dev，避免: 36kr/ai-bot.cn）
SkillNav 已有暗色模式支持（deep indigo 主题），这是相对于大部分中文竞品的优势。注意 daily.dev 用户反馈"暗色背景太蓝"的教训，确保暗色模式的色调适合长时间阅读。

### 7. SoftwareApplication JSON-LD 结构化数据（参考: Glama）
Glama 在每个 MCP Server 详情页使用 SoftwareApplication schema，这帮助 Google 展示富媒体搜索结果。SkillNav 的 Skill 详情页应添加此结构化数据，提升 SEO 竞争力。当前仅有 WebSite 级别的 JSON-LD。

### 8. 编辑观点前置（参考: 机器之心 AI Shortlist, HelloGitHub）
SkillNav 定位 "Wirecutter for AI Agent Skills"，编辑策展是核心差异化。可参考机器之心的 "AI Shortlist" 精选模式 + HelloGitHub 的深度中文描述，将 `editor_comment_zh` 在卡片上直接展示（不是只在详情页），让用户在列表页就能感受到"编辑精选"的价值。

### 9. 工具 + 资讯融合（参考: ai-bot.cn "每日AI快讯"）
ai-bot.cn 在工具导航站中嵌入"每日AI快讯"和博客文章，这与 SkillNav 的"资讯翻译(引流) → 工具导航(留存)"飞轮完全吻合。在 Skills 详情页侧边栏可展示相关 Articles，在 Articles 内容中可推荐相关 Skills，形成交叉引流闭环。

### 10. 多维排序选择器（参考: Glama 7 种排序）
Glama 支持 7 种排序方式（搜索相关性/最近使用/添加日期/更新日期/周下载量/GitHub Stars/近期 Stars），覆盖了用户的不同发现意图。SkillNav 当前的排序选项可扩展，增加"最近更新"、"Star 数"、"编辑评分"等维度。

## 竞品共性趋势

### 1. 从"大全"到"精选" — 策展经济崛起
- mcp.so (18,250+) 和 toolai.io (26,871) 走"大全"路线，但用户体验被数量淹没
- HelloGitHub 和 Glama 走"精选+评级"路线，用户信任度和粘性更高
- **对 SkillNav 的启示**: 168 精选 vs 6,447 全量的策略是正确的，进一步强化"精选"体感

### 2. 搜索体验决定留存
- 所有成功的工具导航站都投入大量精力在搜索上
- 从关键词搜索（mcp.so）到语义搜索（Glama Deep Search）到个性化推荐（daily.dev）
- **对 SkillNav 的启示**: 当前 Orama 客户端搜索是 MVP 方案，后续升级到 Meilisearch 是必要的

### 3. 内容 + 工具 = 更强飞轮
- ai-bot.cn 的"工具导航 + 每日快讯"模式、机器之心的"SOTA 数据库 + 文章库"模式、36kr 的"新闻 + JingData 数据库"模式，都证明内容和工具数据的结合能创造更强的用户粘性
- **对 SkillNav 的启示**: Skills + Articles 的双轮驱动是正确方向，关键是做好交叉引流

### 4. 中文开发者工具站的视觉差距
- 大部分中文站（ai-bot.cn、36kr、机器之心）视觉风格偏保守，缺少暗色模式和现代设计
- 英文站（daily.dev、Glama）视觉更现代
- **对 SkillNav 的启示**: 用现代设计语言（deep indigo 主题 + 暗色模式 + 微动画）服务中文开发者，本身就是差异化

### 5. 社区互动仍处早期
- 大部分站点的用户互动都很弱（GitHub Issue 提交、简单评论）
- daily.dev 的 Squads 和游戏化互动是最佳实践
- **对 SkillNav 的启示**: giscus 评论系统是 MVP 起点，后续可考虑"Skill 使用体验分享"等 UGC 功能

### 6. 移动端和多平台分发
- 36kr 有独立 App + 微信公众号 + 知乎等多平台
- 机器之心有微信服务号（SOTA 模型）
- daily.dev 有浏览器扩展 + Web + 移动端
- **对 SkillNav 的启示**: 网站是基座，知乎/掘金/CSDN/公众号/即刻/Twitter 分发策略已规划，关键是执行

### 7. 结构化数据和 SEO 意识参差不齐
- Glama 做到了 SoftwareApplication schema + 详细的元数据
- mcp.so 作为开源模板项目 SEO 相对基础
- 中文站普遍 SEO 意识较弱
- **对 SkillNav 的启示**: SoftwareApplication JSON-LD 是低成本高收益的 SEO 优化，应尽快实施
