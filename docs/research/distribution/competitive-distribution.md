## 二、其他平台评估

### 知乎 — Must

- 用户重合度：**高**
- SEO/外链价值：**极高**（百度搜索权重极高）
- 策略：专栏同步 + 每周回答 1-2 个热门问题
- 一篇优质回答可持续数月带来流量

### 掘金 — Must（后期）

- 用户重合度：**高**（前端/全栈/Node.js）
- Markdown 直接同步，零改编成本
- HelloGitHub 数据显示单篇约 1000 点击

### 即刻 — Must

- 用户重合度：**极高**（AI 从业者、独立开发者）
- 碎片时间发布，互动率高
- idoubi 即刻 1 年涨 8K 粉

### CSDN — Nice-to-have

- 仅用于百度 SEO 占位
- HelloGitHub 数据仅 77 点击（最低）

### B 站 — Skip

- 视频制作成本过高，1 人团队不可持续
- 即使用 AI 辅助，录制和剪辑仍需大量时间

### GitHub Discussions — Must（已有）

- 已通过 giscus 集成，零额外成本

## 三、1 人团队分发效率

### 每周工时预算（6 小时取中间值）

| 环节 | 时间 | 产出 |
|------|------|------|
| 内容生产 | 2.5h | 1 篇精选 + 1 篇工具推荐 |
| 内容改编分发 | 1.5h | 同一内容适配 4-5 个平台 |
| 即刻/X 碎片发布 | 1h | 3-5 条短动态 |
| 互动维护 | 0.5h | 回复评论、知乎问答 |

### AI 辅助自动化

| 环节 | 自动化程度 |
|------|-----------|
| RSS 采集 | **全自动**（已有 sync-articles.mjs） |
| 翻译 | **全自动**（已有 LLM 管线） |
| 公众号排版 | **半自动**（mdnice/文颜） |
| 多平台同步 | **半自动**（OpenWrite / Wechatsync） |

### MVP 渠道组合

```
skillnav.dev → 公众号 + 知乎 + 即刻 + 掘金 + X
```

### 关键原则

1. **"一鱼多吃"**: 5 个平台只需 1 次创作
2. **"日常学习可见化"**: 不是额外做内容，是把学习过程记录下来
3. **"精选优于量产"**: 阮一峰模式
4. **不维护社群**: 社群维护成本极高
