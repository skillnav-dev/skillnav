# 微信公众号分发渠道深度调研报告

**调研日期**: 2026-03-09
**调研目标**: 评估微信公众号作为 SkillNav 内容分发渠道的可行性与执行策略

---

## 一、平台特性与规则

### 1.1 推荐算法机制（2025-2026）

微信公众号的分发逻辑已从"订阅驱动"全面转向"**社交关系 + 兴趣标签 + 行为数据**"三维推荐模型。这对 SkillNav 这样的新号是利好。

| 流量入口 | 占比 | 驱动因素 |
|----------|------|----------|
| 看一看精选 | ~60% | 兴趣标签 + 社交关系 |
| 搜一搜 | ~15% | 关键词密度 + 内容垂直度 |
| 文章底部推荐 | ~12% | 完读率 + 互动数据 |
| 订阅推送 | ~13% | 粉丝关系 |

**2026 年三个核心算法信号**：
- **完读率**：算法非常关注读者是否"读完"文章，这对长文技术内容是挑战
- **互动率**：点赞/在看是基础，优质评论是黄金信号
- **更新频率与推荐周期的平衡**：一篇文章可能持续推荐 7 天，过高的更新频率反而会冲掉正在推荐中的文章

**新号保护期**：注册后 30 天内系统给予额外流量倾斜。

**AI 内容识别**：平台对纯 AI 生成内容的识别率提高，简单复制粘贴的 AI 文章会被限制推荐。

### 1.2 订阅号 vs 服务号选择

| 维度 | 订阅号 | 服务号 |
|------|--------|--------|
| 群发频率 | 每天 1 次（每次最多 8 篇） | 每月 4 次 |
| 消息位置 | 折叠在订阅号文件夹 | 直接显示在聊天列表 |
| 正文外链 | 不支持（仅"阅读原文"） | 认证后支持 |
| API 权限 | 基础（认证后可发布） | 完整（支付/模板消息等） |
| 适用场景 | 内容传播、品牌建设 | 业务服务、用户管理 |
| 注册主体 | 个人或企业 | 仅企业/组织 |

### 1.3 发文频率与字数限制

| 项目 | 限制 |
|------|------|
| 群发次数 | 订阅号每天 1 次 |
| 每次推送文章数 | 最多 8 篇 |
| 文章字数 | 无明确上限（建议 800-5000 字） |
| 标题长度 | 最长 64 字 |
| 摘要长度 | 最多 120 字 |
| 封面大图尺寸 | 900 x 383 px |
| 封面小图尺寸 | 383 x 383 px |
| "发布"功能 | 不占群发次数，一天可发布多次 |

**重要区分**：微信公众号有"群发"和"发布"两种方式。"发布"不占群发次数，一天可发布多次，发布的内容也有机会获得推荐流量。

### 1.4 外链限制与引流策略

**核心限制**：订阅号正文中**不能插入外部链接**。只有开通微信支付的认证服务号才可以在正文中添加外链。

**可用的引流手段**：

| 方式 | 实现 | 效果 |
|------|------|------|
| "阅读原文"链接 | 文章底部"阅读原文"可跳转任意 URL | 点击率普遍偏低（3-8%） |
| 文中引导文案 | "完整代码见 skillnav.dev" | 间接引流 |
| 二维码 | 嵌入站点二维码图片 | 转化率低 |
| 公众号文章内链 | 可链接到自己其他已发布文章 | 提升内部阅读量 |

### 1.5 原创声明规则

**翻译内容不能声明原创。** 微信公众平台明确规定："改编、翻译、注释、整理他人已有创作而产生的作品均不能说是原创"。

| 内容类型 | 能否声明原创 |
|----------|-------------|
| SkillNav 周刊（编辑策展） | 可以 |
| 深度评测/实战文章 | 可以 |
| RSS 翻译资讯 | 不可以 |
| 课程导读 | 可以 |

### 1.6 留言/互动功能

留言功能自 2018 年起对新号关闭，2024 年开始逐步重新开放。建议尽早注册排队。

## 三、技术内容的公众号最佳实践

### 3.1 代码展示方案

| 方案 | 推荐度 |
|------|--------|
| mdnice 代码块 | 首选（语法高亮、手机可横向滑动） |
| 代码截图 | 备选（长代码段） |
| 微信原生代码块 | 不推荐（无语法高亮） |

### 3.2 长文 vs 短文

- **周刊**：控制在 2000-3000 字，保证完读率
- **评测/教程**：3000-5000 字，深度内容获取搜一搜流量
- **翻译快讯**：800-1500 字，高频短文补充日常内容

### 3.3 Markdown 到公众号排版工具链

| 工具 | SkillNav 适配度 |
|------|-----------------|
| **mdnice** | 最高 — 与现有 Markdown 工作流无缝对接 |
| **doocs/md** | 高 — 可自部署 |
| **文颜 MCP Server** | 极高 — 可集成到现有 Claude 工作流 |

## 四、自动化与 API

### 4.1 微信公众号 API 能力

| 接口 | 功能 | 权限要求 |
|------|------|----------|
| 草稿管理 | 新建/获取草稿 | 已认证账号 |
| 发布接口 | 发布草稿 | **仅认证账号** |
| 素材管理 | 上传图片等 | 已认证账号 |

**2025 年变化**：个人主体未认证账号将被回收发布 API 权限。企业订阅号认证（300 元/年）可使用全部 API。

### 4.2 自动化管线路径

```
Phase 1 (立即): Markdown → mdnice → 手动复制到微信后台 (~15min/篇)
Phase 2 (2-3月后): wenyan-mcp → 自动推草稿到微信后台 → 人工审核群发
Phase 3 (6月后): GitHub Actions 定时生成 + 推送草稿
```

## 五、对 SkillNav 的具体建议

- **选订阅号**：需要每天发文能力，服务号每月仅 4 次
- **企业主体注册 + 认证**：确保 API 权限
- **周刊为核心内容**（可声明原创），翻译资讯作补充（不声明原创）
- **"阅读原文"引流**：每篇指向 skillnav.dev 对应页面
- **启动成本**：¥300 认证 + 每周 1-2 小时人力
