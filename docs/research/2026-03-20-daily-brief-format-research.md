# Daily Brief 多平台格式调研
Date: 2026-03-20
Tags: content, distribution, format, benchmark

> 调研目标：对标海外顶级 AI 日报的格式设计，优化 SkillNav Daily Brief 在各平台的分发效果。

---

## 一、调研对象

| 日报 | 订阅量 | 定位 | 调研方式 |
|------|--------|------|---------|
| TLDR AI | 92万+ | 技术+行业全覆盖 | 抓取 archive 页面分析 |
| The Rundown AI | 200万+ | 3 分钟速览 | 抓取 archive + sitemap |
| Superhuman AI | 150万+ | 产品+消费者 | /feed + /p/ 页面 |
| Ben's Bites | 12万+ | Builder/创业 | /archive + /p/ 页面 |
| The Neuron | 50万+ | Morning Brew 风格 | /feed + /p/ 页面 |

## 二、格式拆解

### 2.1 TLDR AI（最佳结构模板）

**核心特征**：结构化 > 个性化，以可扫描性为第一优先级。

```
标题: "TLDR AI 2026-03-19"
副标题: "Xiaomi & Minimax models 🤖, China's OpenClaw 🦞, Anthropic 81k 📊"
         ↑ 三个关键词 + emoji 预览，不完整句，激发好奇

正文分区（emoji 做视觉锚点）:
🚀 Headlines & Launches     ← 3 条，每条 40-70 字摘要
🧠 Deep Dives & Analysis    ← 2 条
👨‍💻 Engineering & Research   ← 3-4 条
🎁 Miscellaneous            ← 2 条
⚡️ Quick Links              ← 4-5 条，最短

每条格式:
  **标题** (阅读时间)    ← 加粗链接
  2-3 句摘要            ← 无多余修饰

总字数: 1,200-1,400 词
赞助内容: 标记 (Sponsor)，放在相关区块首条
CTA: 仅底部一处，"Get the most interesting AI stories..."
```

**设计决策**：
- 每条 40-70 词，不多不少 — 给够信息做"读/跳"决策
- Emoji 只用于区块标题，正文零 emoji — 专业感
- 无个人观点，纯信息传递 — 定位是"AI 界的 Reuters"
- 每条都有原文链接 — 信任感 + 可追溯

**我们可以学的**：
- 副标题用 3 个关键词 + emoji 预览（我们现在缺这个）
- 每条摘要控制在 40-70 词
- Emoji 只用于结构，不用于修饰

### 2.2 The Neuron（最佳语气模板）

**核心特征**：Morning Brew 风格，专业内容 + 人格化表达。

```
标题: emoji 集群 + 口语化标题
      "😼 AI's Biggest Problem Just Changed. Nobody's Ready."

开头: "Welcome, humans" ← 固定问候语，建立人格

正文结构:
  主要故事 (250-300 词叙事) ← 深度分析一个话题
  🧠 Treats to Try           ← 4-5 条，2-3 句描述
  🌐 Around the Horn         ← 单行 bullet points
  📚 AI Skill of the Day     ← 实操教学一个概念
  🧩 Trivia                  ← 互动问答

字体层级:
  H1 28px Lora Bold   ← 主标题
  H2 24px             ← 区块标题
  Body 15px Poppins   ← 正文
  Small 12px          ← 元数据

颜色: 主题橙 #d96d2e 用于 CTA 和强调
```

**设计决策**：
- 模块化可扫描 — 3 分钟扫完或 20 分钟深读
- 口语化点评 — 括号插入语、自嘲、类比
- 互动元素 — 投票、Trivia，降低跳出
- 赞助内容包装成教育资源，非硬广

**我们可以学的**：
- 主故事深度分析（不只是摘要），用叙事而非列表
- 固定问候语建立品牌人格
- "AI Skill of the Day" 这种实操教学区块
- 颜色系统强调热度

### 2.3 Ben's Bites（最佳人格模板）

**核心特征**：一个人的视角，技术深度 + 个人叙事。

```
结构:
  Hero: 标题 + 副标题（一句话钩子）
  Body: 长文教学（含代码块）← 不是摘要，是深度内容
  "What am I building": 个人项目进展 ← 亲密感
  Headlines: 5-6 条外部链接
  My Feed: 推文嵌入
  Afters: 社交互动展示

语气: 技术但不端着
  "pretty empty"（而非 "currently lacks content"）
  "If it was helpful, reply and let me know 😊"

Emoji: 极少，战略性使用
```

**我们可以学的**：
- 主编观点 > 纯信息聚合（我们的"开发者视角"已在做）
- 个人叙事建立信任（"我这周在做什么"）
- 代码块和技术细节不回避

## 三、各平台最佳实践

### 3.1 X/Twitter Thread

**标杆**: @TheRundownAI (200万粉), @bentaborai

| 维度 | 最佳实践 | 我们 V2 | 差距 |
|------|---------|--------|------|
| **Tweet 1 (Hook)** | 单一最劲爆发现，制造悬念 | ✅ "结论出人意料" | OK |
| **Thread 长度** | 3-5 条（非 7-8 条） | 5 条 | OK |
| **每条独立性** | 每条可单独转发 | ✅ | OK |
| **CTA 推文** | 最后一条，简短，含 handle | ✅ | OK |
| **链接位置** | Tweet 2-3 放链接，不放 Tweet 1 | ✅ | OK |
| **数字先行** | 用具体数字开头 "81,000 人..." | ✅ | OK |
| **方法论推文** | 解释"为什么我们这样选题" | ✅ Tweet 4 | 独有亮点 |

**待改进**：
- Tweet 1 尾部的 "4/5 头部日报同时报道了这件事 👇" 不够 native — 应该在叙事中自然带出
- 每条推文末尾不需要空行

### 3.2 微信公众号

**标杆**: AI 工具集、量子位

| 维度 | 最佳实践 | 我们 V2 | 差距 |
|------|---------|--------|------|
| **标题** | 数字 + 悬念（"8 个让我效率翻倍的 AI 工具"） | "用户想要协作，不是替代" | ⚠️ 缺数字 hook |
| **头图** | 必须有，16:9，品牌色 + 大字标题 | ❌ 无头图 | 需补 |
| **摘要引导语** | 2 行内抓住注意力 | ✅ 蓝色引用框 | OK |
| **卡片排版** | 圆角卡片 + 热度标签 | ✅ | OK |
| **配色** | 品牌主色 + 热度分级色 | ✅ 蓝/红/橙 | OK |
| **文章长度** | 1500-2500 字最佳 | ~800 字 | ⚠️ 偏短 |
| **底部 CTA** | 关注/在看/分享三连 | ✅ 品牌签名 | 缺互动引导 |
| **阅读原文** | 必须有，指向网站 | ❌ 无 | 需加 |

**待改进**：
- 缺头图（微信文章没图打开率低 50%+）
- 正文偏短，微信用户期望更深的解读
- 底部缺"点个在看"等互动引导文案
- 需要"阅读原文"引导到 skillnav.dev

### 3.3 小红书

**标杆**: 三花 AI 资讯、慧心 AI

| 维度 | 最佳实践 | 我们 V2 | 差距 |
|------|---------|--------|------|
| **字数** | 200-400 字最佳 | ~350 字 | ✅ OK |
| **首行 Hook** | 🔥 + 反直觉/数字/疑问 | ✅ "81,000 人告诉 Anthropic" | OK |
| **结构** | Emoji 小标题分 3-4 段 | ✅ | OK |
| **Hashtags** | 5-10 个，混合大小标签 | 7 个 | OK |
| **图片** | **必须有图片/配图卡片** | ❌ 无 | 致命缺陷 |
| **CTA** | "关注我获取每日 AI 资讯" | ✅ | OK |
| **语气** | 口语化、惊叹、接地气 | ✅ | OK |
| **禁忌** | 不放外部链接（会被限流） | ✅ 只放 skillnav.dev | ⚠️ 域名也可能限流 |

**待改进**：
- **致命缺陷：没有图片**。小红书是图文平台，纯文字帖几乎没曝光
- 需要生成配图卡片（品牌色背景 + 大字标题 + 3-4 个要点）
- "全文在 skillnav.dev" 可能被限流，改为"评论区留言领取链接"

### 3.4 知乎

**标杆**: 量子位、机器之心专栏

| 维度 | 最佳实践 | 我们 V2 | 差距 |
|------|---------|--------|------|
| **标题** | 分析性标题（"启示"、"意味着什么"） | ✅ "八万人调研的启示" | OK |
| **深度** | 2000-5000 字，有论证 | ~1500 字 | ⚠️ 偏短 |
| **结构** | 问题→分析→结论→延伸 | ✅ | OK |
| **引用源** | 标注原文链接，增加可信度 | ✅ | OK |
| **个人观点** | 必须有判断，不能纯摘要 | ✅ "开发者启示" | OK |
| **互动** | 末尾提问引导评论 | ❌ 缺 | 需加 |
| **专栏一致性** | 固定栏目名、固定格式 | ❌ 缺 | 需建 |

**待改进**：
- 篇幅可以更长，知乎用户愿意读深度
- 末尾加一个引导讨论的问题（"你会选国产模型做 subagent 吗？"）
- 建立固定栏目名（如"SkillNav AI 日报"专栏）

## 四、跨平台格式对比矩阵

| 维度 | X Thread | 微信 | 知乎 | 小红书 |
|------|---------|------|------|--------|
| **长度** | 5 条推文 | 1500-2500 字 | 2000-5000 字 | 200-400 字 |
| **深度** | 中（摘要+观点） | 中高（解读） | 高（分析论证） | 低（速览） |
| **语气** | 直接、数字先行 | 专业、结构清晰 | 分析性、有判断 | 口语、惊叹 |
| **Emoji** | 适度（🔥📊💡） | 仅标题（🔥📰） | 极少 | 大量 |
| **图片** | 可选（提升互动） | 头图必须 | 可选 | **必须** |
| **链接** | Tweet 2-3 | 阅读原文 | 行内链接 | 不放（限流） |
| **CTA** | 关注+链接 | 在看+关注 | 提问引导 | 关注+评论 |
| **更新频率** | 每日 | 每日或隔日 | 每日或隔日 | 每日 |

## 五、SkillNav V3 格式规范

基于以上调研，定义各平台的目标格式。

### 5.1 X Thread 规范

```
Thread 长度: 4-5 条
Tweet 1 (Hook):
  - 单一最劲爆发现，用数字开头
  - 制造信息差（"结论出人意料"/"你可能不知道"）
  - 不放链接
  - 末尾: 👇 或 🧵 引导读下去

Tweet 2 (Main):
  - 展开 hook 的内容
  - 放原文链接
  - 可以有 1-2 个 emoji 做视觉锚点

Tweet 3 (Others):
  - 今天的其他大事，bullet 格式
  - 每条一行，emoji + 标题 + 一句话

Tweet 4 (Our picks):
  - "如果你做 AI Agent 开发" 开头
  - 4 篇深度文章 + skillnav.dev 链接

Tweet 5 (CTA):
  - 品牌签名，简短
  - @skillnav_dev + skillnav.dev
  - 3-4 个 hashtags
```

### 5.2 微信规范

```
标题: 数字 + 核心发现（"81,000 人调研：用户想要 AI 协作，不是替代"）
头图: 品牌色背景 + 大字标题（需模板化）

正文:
  引导语: 2 行蓝色引用框，概括今日看点

  🔥 行业热点（卡片式）:
    每条: 热度标签（红/橙色）+ 标题 + 2-3 句解读
    卡片: 白底圆角 + 1px 边框

  📰 深度精选:
    列表式，每条: ▸ 链接标题 + 灰色描述

底部:
  品牌签名
  "👉 阅读原文获取中文全文"
  "觉得有用？点个「在看」让更多人看到 🙏"

目标字数: 1500-2000 字
```

### 5.3 小红书规范

```
配图: 必须有！品牌色卡片（需模板化生成）
  - 第一张: 标题卡（大字 + 3 个要点）
  - 第二张: 热点详情卡（可选）

文案:
  首行: 🔥 + 数字/反直觉 hook（15 字内）
  正文: 3-4 段，每段 emoji 小标题
  字数: 200-350 字

  不放链接（限流）
  末尾: "想看中文全文？评论区告诉我 👇"

  Hashtags: 7-10 个
  混合: 大标签（#AI工具 #人工智能）+ 小标签（#AI日报 #开发者日常）
```

### 5.4 知乎规范

```
标题: 分析性（"...的启示"、"...意味着什么"、"为什么...值得关注"）

正文结构:
  ## 为什么这件事值得关注
    （背景 + 共识信号解释）

  ## 核心发现
    （1-2-3 结构化要点 + 解读）

  ## 开发者启示
    （对读者意味着什么，可操作的建议）

  ## 今天的其他信号
    （2-3 个次要话题，每个含"开发者启示"）

  ## 深度文章推荐
    （4 篇链接 + 一句话描述）

  ---
  末尾提问: "你怎么看？" 或具体问题
  品牌签名 + skillnav.dev

目标字数: 2000-3000 字
```

## 六、优先级执行建议

| 优先级 | 行动 | 理由 |
|--------|------|------|
| **P0** | 小红书配图卡片模板 | 没图 = 没曝光，致命缺陷 |
| **P0** | 微信头图模板 | 同上，没图打开率低 50% |
| **P1** | X Thread hook 优化 | Tweet 1 决定整个 thread 的展开率 |
| **P1** | 知乎末尾互动问题 | 提升评论率 → 提升推荐权重 |
| **P2** | 微信互动引导文案 | 提升"在看"→ 提升分发 |
| **P2** | 小红书评论区引导 | 替代链接引流 |

## 七、竞品差异化总结

我们 vs 顶级日报的 **独有优势**：

1. **热度交叉验证** — "4/5 日报报道" 标签，没有任何竞品这么做
2. **开发者视角点评** — 不只是"发生了什么"，而是"和你有什么关系"
3. **中文全文链接** — 热点速览 + 深度阅读一站解决
4. **多平台一次生成** — admin 后台一键复制各格式

我们的 **核心劣势**：
1. **无配图** — 小红书和微信的致命短板
2. **无互动机制** — 知乎缺提问，微信缺"在看"引导
3. **品牌人格弱** — 缺少固定问候语、个人叙事等人格化元素
