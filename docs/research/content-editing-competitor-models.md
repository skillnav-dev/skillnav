# 竞品内容加工模式调研（2026-03-11）

## 行业三种模式

| 模式 | 定义 | 代表 | 人力成本 | 质量 |
|------|------|------|----------|------|
| **纯翻译** | 忠实原文，保留原文结构 | 掘金翻译计划、InfoQ Open-Translating | 低 | 中等 |
| **编译** | 保留核心信息，重写标题/导读，删减/重组段落 | 36氪神译局、机器之心 | 中 (1-2h/篇) | 较高 |
| **改写/原创编译** | 多源综合，以原文为素材重新创作 | 品玩、36氪深度 | 高 (3-5h/篇) | 最高 |

**关键结论**: 纯翻译在 LLM 时代无竞争壁垒，行业主流正向"编译"迁移。

## 各竞品模式详情

### 机器之心 — 编译模式标杆
- 标注体系: "选自 XXX | 编译 | 机器之心 | 编辑 | Panda"
- 加工: 重写中文标题（信息密度更高）+ 导读摘要 + 术语统一
- 资产: 维护 AI 术语翻译数据库 AITD (github.com/jiqizhixin/AITD)

### InfoQ 中文站 — 社区编辑双轨制
- 试译评级 (A/B/C) → 加入团队 → 认领翻译
- 硬扣分项 (意思不符、术语错误) + 软扣分项 (流畅度、格式)
- 偏"忠实翻译"，编辑加工环节较弱

### 掘金翻译计划 — 志愿者模式
- GitHub PR 驱动，1500+ 译者，4000+ 译文
- 优: 覆盖广、成本低；劣: 时效差、质量不一致
- 不适合 SkillNav（人力不够启动社区），但校对 checklist 可参考

### 少数派 — 严格编辑加工
- 针对原创投稿而非翻译
- 公开创作手册 (manual.sspai.com)
- 风格指南（段落结构、中英混排、标点规范）可直接参考

### 36氪神译局 — 编译模式标杆
- 标准格式: 改写标题 + 编者按 + 编译正文 + 原文标注
- 标题风格: 信息密度高、口语化、吸引点击

## 高质量编译的 7 步流程

1. **选题判断** — 原文质量 + 读者匹配 + 时效性 (半自动)
2. **基础翻译** — 全文翻译 (全自动)
3. **标题改写** — 重写而非翻译，加信息钩子 (全自动+审核)
4. **导读/编者按** — 1-3 句核心价值 + 来源说明 (全自动+审核)
5. **结构优化** — 长段落拆分 + 加小标题 + 删减无关内容 + 补充中国语境 (大部分自动)
6. **语言润色** — 消除翻译腔 + 中英混排规范 (全自动)
7. **质量终审** — 事实核查 + 链接有效性 + 通读流畅度 (必须人工)

## LLM 自动化可行性

| 步骤 | 自动化程度 | 说明 |
|------|-----------|------|
| 选题判断 | 半自动 | LLM 评分，人工决策 |
| 基础翻译 | 全自动 | 已实现 |
| 标题改写 | 全自动+审核 | LLM 生成候选，人工选择 |
| 导读/编者按 | 全自动+审核 | LLM 生成，编辑观点需人工 |
| 结构优化 | 大部分自动 | 段落/标题可自动，本地化需人工 |
| 语言润色 | 全自动 | 消除翻译腔是 LLM 强项 |
| 质量终审 | 必须人工 | 事实核查、语境判断 |

**效率**: 传统编辑 2-4h/篇 → LLM 辅助后 15-30min 人工。

## 可参考的 Prompt 模板

**宝玉三步翻译法** (少数派验证有效):
1. 直译 — 忠实原文
2. 反思 — 指出直译问题
3. 意译 — 基于反思重写

完整实现: github.com/OpenAI-Translator, eallion/52aadf05e 的 prompt gist。

## 对 SkillNav 的建议

1. 走"编译"模式而非"纯翻译" — 标题改写 + 导读 + 结构优化
2. 最高 ROI 加工步骤: 标题改写 > 导读/编者按 > 术语统一 > 结构优化
3. 人工集中在: 选题判断 + 质量终审 + 编辑观点
4. 术语表是长期资产 — 参考机器之心 AITD
