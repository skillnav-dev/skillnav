# 编辑标准体系缺口分析（2026-03-11）

基于 4 份现有规范的全面审查。

## 已建立的标准

### 质量评估
- **双维度评分**: 开发者价值 (DV) + 读者价值 (RV)，综合分 = max(DV, RV)
- **准入判断**: "读完能**做**点什么"才收，纯新闻快讯不收
- **自动处置**: score ≥3 → published, =2 → draft 待审, =1 → hidden
- 来源: content-pipeline-spec.md §三

### 发布决策
- score ≥5 + 翻译质量好 → published
- score ≥5 + 翻译有瑕疵但核心准确 → published + 编辑注
- 特别优质 + 与 Skills/MCP 强相关 → published + is_featured=true
- 来源: content-operations-spec.md §5.2

### 内容层级
- **translated**: RSS 自动采集 + LLM 翻译，管线全自动
- **editorial**: AI 辅助 + 主编策展/原创（周刊、评测、月度盘点）
- 来源: content-strategy-v2.md §三

### 分发门槛
- skillnav.dev: 综合分 ≥3
- 知乎/掘金: 综合分 ≥4
- 公众号群发: 综合分 =5 或主编精选
- 来源: content-pipeline-spec.md §4.1

## 缺失的环节（5 个关键缺口）

### 缺口 1: translated 层的"编辑加工"流程
- **现状**: 审核只做"发布/隐藏"二元决策
- **缺失**: 没有"对翻译文章进行改写/润色后再发布"的路径
- **影响**: 高分但翻译质量一般的文章无法被提升

### 缺口 2: editorial 层的原创写作规范
- **现状**: 周刊有完整工作流和模板
- **缺失**: 原创评测/分析类文章无选题标准、写作模板、质量标准

### 缺口 3: translated → editorial 的升级路径
- **现状**: 两个层级互相独立
- **缺失**: "发现一篇高分翻译文章，值得深度加工成编辑精选"的触发条件和加工流程

### 缺口 4: 分发版本的改编 SOP
- **现状**: 各平台字数/格式要求已定义
- **缺失**: 谁来改编、用什么工具、改编后如何审核

### 缺口 5: 翻译质量的评判标准
- **现状**: 发布决策提及"翻译质量好" vs "有瑕疵"
- **缺失**: 没有具体的质量维度定义（术语准确度、流畅度、格式保留度等）
